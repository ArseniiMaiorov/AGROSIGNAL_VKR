# Архитектура (этапы 1-2)

- `backend`: Python API, миграции PostGIS, unit/integration тестирование, документация.
- `frontend`: каркас каталогов SPA (заполнение на следующих этапах).
- `docs`: общая проектная документация и протоколы.

Текущий runtime-стек backend:
- API: Python HTTP-сервис (`/health`)
- БД: PostgreSQL + PostGIS
- Очереди/кэш: Redis
- Оркестрация: Docker Compose

Нефункциональные требования:
- При занятом целевом порте система автоматически выбирает свободный порт и сохраняет его в `backend/.api_port`.

Модель данных этапа 2:
- `enterprises` — предприятия.
- `roles`, `app_users` — пользователи и роли.
- `crops`, `seasons` — культуры и сезоны.
- `fields` — поля (полигон в PostGIS) с вычисляемой площадью `area_ha`.
- `work_journal` — дневник работ.

Геовалидаторы для `fields.geom`:
- только `Polygon`;
- SRID строго `EPSG:4326`;
- запрет пустых и невалидных геометрий;
- отдельная ошибка для самопересечения.
