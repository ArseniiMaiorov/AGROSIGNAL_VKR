# Архитектура (этап 1)

- `backend`: Python API, модульная структура, тесты, миграции и документация.
- `frontend`: каркас каталогов SPA (будет заполняться на следующих этапах).
- `docs`: общая проектная документация и протоколы.

Текущий runtime-стек backend:
- API: Python HTTP-сервис (`/health`)
- БД: PostgreSQL + PostGIS
- Очереди/кэш: Redis
- Оркестрация: Docker Compose

Нефункциональные требования (уточнено):
- При занятом целевом порте система должна автоматически выбирать свободный порт и явно сообщать его в логах/документации запуска.
