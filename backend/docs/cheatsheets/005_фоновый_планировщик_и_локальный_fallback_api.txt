Фоновый планировщик и локальный fallback API
Это эксплуатационный механизм, который удерживает систему в рабочем состоянии даже при проблемах локальной Docker-сети.

Что это такое:
- `stage_scheduler.py` запускает по расписанию цикл синхронизации, экспорта и TTL-проверки.
- `make up` проверяет доступ к API через published port; если порт недоступен, API автоматически поднимается локально на хосте.

Зачем нужно:
- Синхронизация источников выполняется автоматически в фоне, без ручного запуска команд.
- При сетевом блокере Docker портов пользователь всё равно получает рабочий HTTP API на локальном порту.

Вход:
- Переменные окружения планировщика (`STAGE_SCHEDULER_INTERVAL_SECONDS`, `STAGE_SCHEDULER_SYNC_HOURS`, `STAGE_SCHEDULER_RETENTION_DAYS`).
- Команда запуска `make up`.

Выход:
- `API_MODE=container` или `API_MODE=local` в `backend/.api_mode`.
- Файлы PID/логов: `.api_local.pid`, `.api_local.log`, `.scheduler.pid`, `.scheduler.log`.
- Доступный endpoint `http://127.0.0.1:<API_PORT>/health`.

Влияние на итог:
- Этапы 1-4 остаются воспроизводимыми и проверяемыми в среде с нестабильным Docker networking.
