Healthcheck API и базовый контур проекта
Это стартовая техническая основа системы, которая подтверждает, что backend запущен и отвечает.

Что это такое:
- Endpoint `/health`, возвращающий состояние сервиса и служебные метаданные.

Зачем нужно:
- Быстрая проверка, что API в рабочем состоянии.
- Основа для healthcheck контейнера и мониторинга.

Вход:
- HTTP GET запрос без параметров.

Выход:
- JSON со статусом, именем сервиса, окружением и временем ответа.

Влияние на итог:
- Позволяет воспроизводимо проверять запуск проекта и фиксировать готовность инфраструктуры на раннем этапе.
