API v1 и пользовательские сценарии без фронта
Это прикладной слой backend, который позволяет пройти весь основной пользовательский путь через HTTP API без интерфейса Vue.

Что это такое:
- Набор endpoint'ов `/api/v1/...` для домена (предприятия, поля, культуры, сезоны, операции), данных (погода/спутник/синхронизация), помощника агронома и экспорта.
- Единый контракт ответов/ошибок с `api_version` и `request_id`.
- RBAC, аудит действий и метрики запросов.

Зачем нужно:
- Подключение фронтенда становится прямой задачей интеграции по стабильному контракту.
- Сценарии пользователя проверяются и воспроизводятся даже без UI.
- Ошибки, права доступа и статус данных прозрачны и диагностируемы.

Вход:
- HTTP-запросы к `/api/v1/...` c заголовком `X-User-Email`.
- Для повторяемых операций (`sync/run`, `export`) — `Idempotency-Key`.
- Параметры диапазона (`from/to`), гранулярности, фильтров и пагинации.

Выход:
- Для успеха: `data + meta` (`api_version`, `request_id`, дополнительные метаданные).
- Для ошибки: `error.code`, `error.message` (на русском), `error.details`, `request_id`.
- Для пустых диапазонов: `200 + meta.status=NO_DATA` и пустые массивы.

Влияние на итог:
- API закрывает Definition of Done этапа 5: CRUD, данные, помощник, экспорт, RBAC, аудит, наблюдаемость.
- Этапы 1-4 не ломаются (регрессионные тесты PASS).
- Проект готов к подключению Vue SPA по готовому HTTP-контракту.
